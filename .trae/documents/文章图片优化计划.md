## 新增带图片文章的详细流程

### 方式一：从 Obsidian 导入（强烈推荐！）

#### 0. Obsidian 前期设置
- 在 Obsidian 设置中：文件与链接 → 新链接格式 → 选择“相对路径”
- 确认 Obsidian 的图片附件文件夹设置（可以单独设置）

#### 1. 在 Obsidian 中完成创作
- 文章写好，图片直接插入（Obsidian 自动管理）

#### 2. 使用导入脚本
```bash
# 运行 Obsidian 导入脚本
python scripts/import_obsidian.py [你的Obsidian文章路径.md] [类型]

# 示例
python scripts/import_obsidian.py ~/Obsidian/Vault/My\ Article.md article
python scripts/import_obsidian.py ~/Obsidian/Vault/Review.md review
```

#### 3. 脚本自动处理：
- 自动提取 Frontmatter 信息
- 自动解析 Markdown，识别所有 `![[image.png]]` 或 `![...](...)` 格式的图片
- 自动定位 Obsidian vault 中的图片文件
- 自动复制所有图片到 `assets/images/[类型]/[slug]/`
- 自动将图片路径替换为项目标准格式
- 自动生成最终文章到 `_articles/` 或 `_reviews/` 目录

#### 支持的图片格式：
```markdown
<!-- Obsidian 标准格式 -->
![[image.png]]

<!-- 标准 Markdown 格式 -->
![图片说明](../attachments/image.png)
```

#### 4. 构建预览
```bash
python scripts/build.py
cd _site && python -m http.server 8080
```

### 方式二：手动创建（推荐新手）

#### 1. 准备素材
- 文章内容（Markdown格式）
- 准备图片文件

#### 2. 创建文章文件
```bash
# 复制模板文件位置
docs/ARTICLE_TEMPLATE.md

# 创建新文章到 _articles/ 或 _reviews/ 目录
```

#### 3. 准备图片
```
# 图片目录结构：
assets/images/
├── articles/
│   └── [文章slug]/
│       ├── featured.png      (特色图片，可选)
│       ├── image1.png
│       └── image2.png
└── reviews/
    └── [评测slug]/
        ├── featured.png
        └── ...
```

#### 4. 编写文章内容
```markdown
---
title: "文章标题"
subtitle: "文章副标题"
description: "SEO描述"
date: 2025-05-22
category: "分类"
tags: ["标签1", "标签2"]
reading_time: 10
author: "作者名"
lang: zh
layout: article  # 或 review
featured_image: "assets/images/articles/[slug]/featured.png"  # 可选
---

## 正文内容...

![图片说明](assets/images/articles/[slug]/image1.png)

继续正文...
```

#### 5. 构建预览
```bash
# 构建网站
python scripts/build.py

# 本地预览
cd _site && python -m http.server 8080
```

## 图片使用规范

### 格式与尺寸
- **特色图片：建议 1200x630 (16:9 比例)
- **内容图片**：最大宽度 800-1200px
- **格式**：PNG/JPG/WebP
- **大小**：单张图片 < 500KB
- **文件命名**：全小写，用连字符分隔

### Markdown 引用方式
```markdown
<!-- 基础引用 -->
![图片描述文字](assets/images/[类型]/[slug]/[文件名].png)

<!-- 带说明 -->
```

## 注意事项

- 保持项目纯 HTML/CSS/JS 架构
- 不引入额外依赖
- 保持构建过程简单
- 图片使用相对路径，确保GitHub Pages兼容性
- 保持响应式设计
- 图片命名全小写，使用连字符
- 图片路径在文章内引用路径与实际存放路径一致
