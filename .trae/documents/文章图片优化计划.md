
# 文章图片优化计划

## 概述

为项目添加图片支持，包括创建图片存储结构、更新模板样式、新增带图文章、简化文档流程。

## 当前项目结构分析

- 纯 HTML/CSS/JS 项目，无框架
- 使用自定义 Python 构建脚本 (`scripts/build.py`)
- 当前文章存放在 `_articles/` 目录
- 模板存放在 `_layouts/` 目录
- 没有图片存储或处理机制

## 实施方案

### 1. 创建图片存储目录

- 创建 `assets/images/` 目录
- 创建子目录 `articles/` 和 `reviews/` 分别存放不同类型文章的图片

### 2. 更新构建脚本 (`scripts/build.py`)

- 添加图片资源复制功能
- 将图片从 `assets/images/` 复制到 `_site/assets/images/`
- 保持目录结构不变

### 3. 更新文章模板 (`_layouts/article.html`)

- 添加特色图片显示区域
- 优化图片显示样式
- 添加图片说明文字支持
- 保持响应式设计

### 4. 更新样式文件 (`css/article.css`)

- 添加文章图片样式
- 添加特色图片样式
- 添加图片说明样式
- 优化移动端图片显示

### 6. 创建文章模板文档

## 注意事项

- 保持项目纯 HTML/CSS/JS 架构
- 不引入额外依赖
- 保持构建过程简单
- 图片使用相对路径，确保GitHub Pages兼容性
- 保持响应式设计

## 风险处理

- 图片加载失败：使用alt文字作为后备
- 图片尺寸过大：建议优化图片尺寸在合理范围内
- 目录结构变更：确保向后兼容
